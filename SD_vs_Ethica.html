Comparing Ethica GPS data to SenseDoc GPS data¶

See also analyses.ipynb, which focuses on comparing location detection using classical GPS tracks (SenseDoc) to Ethica GPS track, yet contains a series of preanalyses to try to understand why we get results so different.

Objective: assess the validity of Ethica Data as an alternative GPS data source for location detection¶

  1. assess spatial coincidence of both GPS tracks
  2. assess similarity of GPS tracks at various spatial and temporal scales

Clean Ethica GPS track¶

Ethica track has been cleaned according to Mohammad suggestion:

  • filter fix with accu < 100m
  • sync GPS fix with duty cycle (extracted from battery data)
  • resync timestamp to smartphone time or satellite time, depending on fix being reused (i.e. satellite time outside DC) or not
  • when more than one fix exists for the same timestamp, keep only the one with the best accuracy

Comments:

  • The two tracks don't seem to be temporally synchronized, as the distance between a SD GPS fix and the corresponding Ethica one for the same timestamp is generally over 2km
  • The best performance (i.e. closest fixes) generally happens at the beginning of the DC, then it degrades and stays more or less constant around 2km~2,5km for the remaining of the DC
  • Out of DC fixes (i.e. reused) at the beginning of DC exhibit the best performance (~140m); this is probably due to the fixes being reused correspond to home location, which is corroborated by the very high number of cases (almost 130000 cases)

Similarity of SD and GPS tracks using Cross-correlation¶

Using cross-correlation to assess similarity of both tracks, at various temporal and spatial scales. Only the common pportion of the survey is considered. 2D histograms are computed and compared using cross-correlation to measure the similarity.

# Compute cross-correlation coeeficient of two GPS tracks
def compute_cc_coeff(trk1:np.array, trk2:np.array, cell_size:int, aoi:np.array=None, clip_extent:np.array=None) -> tuple:
    """
    Compute the 2D cross-correlation of tracks 1 and 2. Track coordinates should be specified in planimetric units
    --
    Parameters:
    - trk1: a 2D array of X and Y coordinates ([1, 2, 2, 4], [-45, -44, -44, -43]) describing the first track
    - trk2: same as trk1, for the second track
    - cell_size: the size of the raster cells, expressed in the same unit as the track coordinates
    - aoi: a 2D array of LL/UR corners [[xmin, xmax], [ymin, ymax]] defining the area of interest;
        if None, the union of track 1 and 2 extents will be used
    - clip_extent: a 2D array of LL/UR corners [[xmin, xmax], [ymin, ymax]] defining the max extent of AOI;
        if None, no constraint is enforced
    Returns:
    - a tuple with (histogram1, histogram2, cross-correlation)
    """
    if trk1.size == 0 or trk2.size == 0:
        return (None, None, None)

    # Compute 2D histograms from tracks
    trk1_x, trk1_y = trk1
    trk2_x, trk2_y = trk2
    
    if aoi is None: # no AOI given, compute it from track extents
        x = np.concatenate([trk1_x, trk2_x])
        y = np.concatenate([trk1_y, trk2_y])
        aoi = [[min(x), max(x)], [min(y), max(y)]]

    if clip_extent is not None: # a max extent is given, check constraint vs. AOI
        aoi = np.maximum(aoi, clip_extent, where=[[True, False], [True, False]], out=np.minimum(aoi, clip_extent))

    # Check array size
    max_size = 5000*5000
    aoi_x, aoi_y = aoi
    nbins_x = int(np.ceil((aoi_x[1]-aoi_x[0])/cell_size))
    nbins_y = int(np.ceil((aoi_y[1]-aoi_y[0])/cell_size))
    n_cells = nbins_x * nbins_y
    if n_cells > max_size:
        print(f'Warning: raster size over limit {n_cells} > {max_size}')
    
    hist1, _, _ = np.histogram2d(trk1_x, trk1_y, bins=[nbins_x, nbins_y], range=aoi) #, density=True)
    hist2, _, _ = np.histogram2d(trk2_x, trk2_y, bins=[nbins_x, nbins_y], range=aoi) #, density=True)

    # Compute cross-correlation of z-histograms
    cc = correlate2d((hist1-np.mean(hist1))/np.std(hist1), (hist2-np.mean(hist2))/np.std(hist2), mode='valid')
    cc = np.max(cc) / nbins_x / nbins_y

    return (hist1, hist2, cc)

# Wrapper function to compute CC
def wrapper_cc(iid, cellsize, start=None, end=None, epoch='', table='cross_corr_sd_eth', schema=None) -> tuple():
    # For multiprocessing, see https://docs.sqlalchemy.org/en/13/core/pooling.html#pooling-multiprocessing
    engine.dispose()

    # Get start/end if not given
    if start is None:
        start = surveys.loc[surveys['iid'] == iid, 'survey_start'].dt.to_pydatetime()[0]
    if end is None:
        end = surveys.loc[surveys['iid'] == iid, 'survey_end'].dt.to_pydatetime()[0]
    timestep = end - start

    # Get data
    sd_data = pd.read_sql(text(sql_sd), engine, params={'iid': iid, 'survey_start': start, 'survey_end': end})
    eth_data = pd.read_sql(text(sql_eth), engine, params={'iid': iid, 'survey_start': start, 'survey_end': end})

    # Compute cross-correlation
    aoi = aoi_dict.get(str(iid)[0], None)
    h1, h2, cc = compute_cc_coeff(np.array([sd_data.x, sd_data.y]), np.array([eth_data.x, eth_data.y]), cellsize, clip_extent=aoi)

    # Save data to DB
    if h1 is not None:
        h1 = h1.tolist() # psycopg2 cannot store numpy array, need to convert to list first
    if h2 is not None:
        h2 = h2.tolist() # psycopg2 cannot store numpy array, need to convert to list first
    cc_results = pd.DataFrame.from_records([(epoch, iid, cellsize, timestep, h1, h2, cc)], \
        columns=['epoch', 'iid', 'cellsize', 'timestep_ns', 'histogram1', 'histogram2', 'cccoeff'])
    cc_results.to_sql(table, engine, schema, if_exists='append', index=False)
    

def wrapper_mp(params:list):
    """Multiprocessing wrapper func; expecting params as list of tuples"""
    with mp.Pool() as pool:
        results_mp = [pool.apply_async(wrapper_cc, p) for p in params]

    return [r.get() for r in results_mp]

Discussion¶

Fig 1a and 1b show the distribution of the percentage of area of overlap between Ethica and SenseDoc convex hulls. When both convex hulls are identical, both values = 100% (upper right corner of Fig 1a.). Fig. 2a and 2b show the distribution of Ethica convex hull areas vs. SenseDoc ones. Although it seems that some level of correlation exists between both variables (aligned points in scatterplot), the correlation coeff clearly discards this possibility (corr = 0.10)

Median area of Ethica track convex hull = 75.3 sq.km while SenseDoc = 100.4 sq.km. Except for some outliers, Ethica convex hulls tend to be smaller than SenseDoc ones, hence overlapping area tends to cover a larger portion of Ethica convex hulls than SD ones (more points clamped to the right side of the scatterplot in fig. 1a and 1b). No clear viusal trend when looking at duration of survey (fig. 1a) nor size of the overlapping area (fig. 1b).

Similarly for CMA bound activity space, Fig 3a and 3b focus on % of overlap area compared to Ethica convex hull area (X) vs. compared to SD convex hull area (Y); distribution pattern more or less looks like unbound results. Conversely, Fig 4a and 4b show a clearer relation between both convex hull areas in CMA bound compared to unbound data. This is comforted by the correlation coeff (= 0.75). Median area of Ethica track convex hull = 66.46 sq.km while SenseDoc = 90.10 sq.km.

Results¶

Only the common period for Ethica and SenseDoc data are considered in the results below. Buffer of 250m and 500m radius have been computed around the GPS points. Another series of buffer of same radius have also been computed along tracks, i.e. path segments bweteen GPS points as long as the apparent speed of movement of the path remained below 140km/h.

Dynamic time warping¶

DTW allows to assess the resemblance of two time series by locally stretching and/or compressing one to make it look alike the reference one. Here, SenseDoc GPS tracks are considered the reference while Ethica tracks are dynamically warped to match them. The normalized distance between the two tracks after alignment is then computed.
